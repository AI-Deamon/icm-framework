# Unified Creative-Technical-Business Workflow

**Source:** Skool: The Foundation / Module 4 / 4.3 Claude Desktop as a Thinking Partner

## Core Idea

Creative work, technical work, and business work are the same problem wearing different clothes. AI collapses these silos — the same folder system, the same context files, the same workflow works for writing copy, coding a feature, and planning a launch.

## Why It Matters

- Before AI, you needed different consultancies for creative (design agency), technical (dev shop), and business (strategy firm). With AI, one person with good context files can operate across all three — because AI handles the execution layer in each domain.
- Jake's demonstration: one project viewed through three lenses. The creative lens (what story are we telling?), the technical lens (how do we build it?), the business lens (what value does it create?). Same project, same folder, different CONTEXT.md files for each lens.
- This only works if your folder system is domain-agnostic. If you designed it just for coding, it won't work for creative. The key is abstract context files that describe principles, not procedures.

## How I'll Apply It

Structure my context files around principles (why), not procedures (how). A principle-based CONTEXT.md works for any domain. A procedure-based one only works for one task. When switching from coding to writing to planning, the same folder structure should still guide the AI.

## Cross-Reference

This is the folder workspace system applied beyond coding. Connects to the template pattern (Foundation & Mindset) — principles are templates, procedures are filled-in slots.
