# Quick Start: First Folder Setup

**Source:** Skool: The Foundation / Module 1 / 1.1-1.3 Quick Start

## Core Idea

The on-ramp to the entire ICM system: three files in five minutes that change how Claude responds to you. No theory, no full architecture — just a working folder that ends the "start from zero every time" pattern.

## Why It Matters

- The three-file minimum: AGENTS.md (identity + rules), CONTEXT.md (current project, quality standards, what to avoid), REFERENCES.md (examples, links, background material).
- Three ways to point Claude at the folder: (1) Claude Code in terminal — `cd my-folder; claude` reads files automatically, (2) Claude.ai Projects — upload as Project Knowledge, (3) Copy-paste into first message.
- 1.3 teaches a five-part prompting framework: Identity, Context, Instruction, Format, Constraints — works for writing, coding, and research. This is the prompt-side counterpart to the folder-side structure.
- "You gave Claude three things it never has in a normal chat: who you are, what the project is, and what good looks like."

## How I'll Apply It

For every new project, start with the three-file minimum before doing anything else. Don't worry about perfect structure — the files grow with you. Edit when the project changes, add more files when Claude needs to know more, create separate folders for separate projects.

## Cross-Reference

This is the on-ramp to the full three-layer architecture in Module 3.1. The five-part prompt framework (1.3) connects to the template pattern (2.3) — prompts are templates with slots.
