# Building Websites with Claude Code

**Source:** Skool: Implementation Playbooks / Module 3 / 3.1-3.3 Building Websites

## Core Idea

A complete workflow for building real websites with Claude Code and deploying them free on GitHub Pages — in under 10 prompts, under 20 minutes. The folder structure (planning → build → deploy) is the same three-layer system applied to web development.

## Why It Matters

- The workflow: (1) Analyze a reference site (take screenshots, let Claude study them), (2) Create a briefing doc in the planning folder, (3) Build incrementally with Claude Code, (4) Deploy to GitHub Pages for free.
- GitHub Pages is free hosting. GitHub Actions can automate deployment. The only cost is the Claude subscription. No Wix, no Squarespace, no monthly hosting fees.
- Module 3.2 slows down on how GitHub works: repos, branches, pushing, Pages, Actions. The insight: clean folder structure saves tokens because Claude reads a single AGENTS.md instead of guessing the project structure.
- Module 3.3 teaches the importance of prompt sequencing — don't ask Claude to build the whole site at once. Start with the brief, then the layout, then components, then content, then polish.

## How I'll Apply It

For my next personal site or client site, use this exact pipeline: planning workspace → briefing doc → incremental builds with Claude Code → GitHub Pages deploy. The folder system IS the project management tool — no Trello, no Notion, just folders and markdown files in the project.
