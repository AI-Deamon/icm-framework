# OpenClaw: Orchestration Over AI

**Source:** Skool: The Foundation / Module 2 / 2.5 OpenClaw Has 350K+ Stars

## Core Idea

OpenClaw has 350,000+ GitHub stars and zero AI inside it. It's 90% orchestration (routing logic, traditional code) and 10% AI API calls pushed externally. The lesson: value lives in the architecture around AI, not the AI itself. OpenClaw is the highway system; you bring your own car (AI model).

## Why It Matters

- The 60/30/10 pattern emerges here: ~60% traditional code (platform integrations, file handling, networking), ~30% rule-based logic (routing decisions, security, management), ~10% actual AI processing. If you only look at the 10%, you miss 90% of the system.
- Three diagnostic questions for any AI tool: (1) Where is the intelligence? (2) Where is the value? (3) What gets commoditized next?
- "Value lives above whatever just got commoditized. Always." This is the same pattern as the abstraction ladder — the AI model itself is the commodity; the orchestration layer is where value accumulates.
- The "Sherlocked" pattern: when big platforms build what you built natively, your product disappears. Orchestration survives because it coordinates, not because it generates.

## How I'll Apply It

When evaluating any AI tool or building my own, ask: "What percentage is actual AI vs. orchestration?" If I'm building the orchestration layer, that's durable. If I'm building on the AI's capabilities alone, that's temporary. My folder structure IS my personal orchestration layer — I'm the conductor, Claude is the musician.
