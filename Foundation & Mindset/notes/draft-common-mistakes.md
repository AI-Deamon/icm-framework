# Common Mistakes in Folder Architecture

**Source:** Skool: The Foundation / Module 3 / 3.3 Common Mistakes and How to Fix Them

## Core Idea

Seven specific mistakes people make setting up the folder architecture — drawn from real community experience — each with a clear fix. They all point in one direction: keep the system small, focused on the work, and updated as you go.

## Why It Matters

1. **AGENTS.md too long** — fix: fit on one screen (40-50 lines max). Identity, folder structure, routing table, naming conventions only. Everything else goes in workspace CONTEXT.md files.
2. **Skipping the routing table** — fix: three columns (Task, Go to, Read). One row per work type. Add a 4th Skills column for Layer 3.
3. **Too many workspaces** — fix: start with 2-3. "If you're not sure whether something deserves its own workspace, it does not." The mental mode test decides.
4. **Context files describe what AI should BE instead of what the WORK is** — fix: 80% describing the work (project, audience, quality standards, what to avoid), 20% or less on behavioral instructions. "If it reads like a project brief a new team member could work from, you're in the right place."
5. **Never updating context files** — fix: "Last updated" line at top. The single highest-leverage habit in the whole system.
6. **Everything in one flat folder** — fix: more than 8-10 files at the same level needs subfolders.
7. **Building the whole system before using it** — fix: "Your first version should take 15 minutes. If it took longer, you over-built." Build minimum, start working, add based on real needs.

## How I'll Apply It

Run an audit on my current Framework folder against these 7 mistakes. Especially check: is my AGENTS.md under 50 lines? Do I have a routing table? Am I describing the work (not telling the AI what to be)? When was the last time I updated context files?
